# プロジェクト開始ガイド

このテンプレートを使用して新しいWeb制作プロジェクトを開始するための標準手順書です。

## 1. プロジェクトの準備（クローンと初期化）

まず、任意の作業ディレクトリでテンプレートをクローンし、新しいプロジェクトとして初期化します。

### 1-1. テンプレートをクローン（[project-name] は実際の案件名に変更）

```bash
git clone https://github.com/crawlre/vite-template.git [project-name]
```

### 1-2. ディレクトリに移動

```bash
cd [project-name]
```

### 1-3. Gitの履歴をリセット（テンプレートの履歴を切り離す）

```bash
rm -rf .git
git init
```

### 1-4. 依存パッケージのインストール

```bash
npm install
```

## 2. 開発環境の起動

### ローカルサーバーの起動（ブラウザが自動で開きます）

```bash
npm run dev
```

## 3. コーディング準備（カンプ分析と設定）

デザインカンプを元に、共通のスタイルを定義します。

### 3-1. グローバル変数の設定

`src/scss/global/`内の各ファイルに、カンプから抜き出した値を反映させます。

`_color.scss`: メインカラー、テキストカラー、背景色など。

`_font.scss`: 基本フォントサイズ、フォントファミリーなど。

### 3-2. 画像の配置

カンプから書き出した画像を `src/assets/images/` に配置します。

`common/`: 共通パーツ（ロゴ等）

`top`: 各ページ専用

## 4. コーディングの基本ルール

本プロジェクトでは FLOCSS 設計を採用しています。
新しい SCSS ファイルを作成した際は、必ず `src/scss/style.scss` にて 手動で `@use` してください。

### ディレクトリ構成の役割

`Foundation`: リセットCSSや基本のベーススタイル。

`Layout`: ヘッダー、フッター、メインコンテンツなどの大きな枠組み。

`Component`: ボタン、カード、ナビゲーションなど、再利用可能な最小単位。

`Project`: 特定のページに依存する固有のセクション。

`Utility`: 微調整用のクラス（余白の調整など）。

## 5. 納品・ビルド

### 本番公開用ファイルの生成（distフォルダに出力されます）

`npm run build`

## メモ

- コミットは細かく: 「ヘッダー完成」「色変数の定義」など、作業単位で git commit を行うことで、不測の事態に備える。

- READMEの更新: 案件固有の仕様（特殊なプラグインの使用など）がある場合は、このREADMEに追記する。

- 共通パーツのストック: 汎用性の高いコンポーネントが作れたら、元のテンプレートリポジトリにも反映させる。
